3、类加载器.

作用：.

  • 加载Class文件

classpath到底指什么(这边文章写得很详细)：https://segmentfault.com/a/1190000015802324

分类：.

分类来自：https://www.jianshu.com/p/1e4011617650

  • 应用程序加载器 AppClassLoader 定义类编译输出目录(包含资源)
    • java编写，加载程序所在的目录，如user.dir所在的位置的class
  • 扩展类/平台加载器 ExtClassLoader /jre/lib/ext -- JDK 11之前
    • java编写，加载扩展库，如classpath中的jre ，javax.*或者
      java.ext.dir 指定位置中的类，开发者可以直接使用标准扩展类加载器。
  • 启动类/根加载器 BootstrapClassLoader rt.jar
    • c++编写，加载java核心库 java.*,构造ExtClassLoader和AppClassLoader。由于引导类加载器涉及到虚拟机本地实现细节，开发者无法直接获取到启动类加载器的引用，所以不允许直接通过引用进行操作
  • 用户自定义加载器
    • 实现 ClassLoader这个抽象类的类

拓展： java与c++相比，去掉了 指针和 内存管理

public class ClassLoaderTest {

    public static void main(String[] args) {
        ClassLoaderTest test = new ClassLoaderTest();

        Class<? extends ClassLoaderTest> aClass = test.getClass();

        //获取 该Class 的类加载器
        System.out.println(aClass.getClassLoader()); // AppClassLoader@3fee733d

        // 获取其 父类加载器
        System.out.println(aClass.getClassLoader().getParent()); // PlatformClassLoader@3fa77460 （JDK1.9之前 为ExtClassLoader）

        // 获取其 父类的父类记载器
        System.out.println(aClass.getClassLoader().getParent().getParent()); // null 不是没有父加载器，而是无法获取 BootstrapClassLoader
    }
}

4、双亲委派机制.

为了保证安全

当某个类加载器需要加载某个.class文件时，它首先把这个任务委托给他的上级类加载器，递归这个操作（直到启动类加载器），如果上级的类加载器没有加载，自己才会去加载这个类。

大致可以划分为4步

  1. 类加载器接收到类加载的请求 对应加载器AppClassLoader
  2. 他会将这个请求委托给父类加载器，一致往上委托，直到启动类加载器 对应加载器 BootstrapClassLoader
  3. 加载器检查 能否加载 这个类 （此时加载器 先是启动类，然后其子类）
    1. 如果能，就结束，使用当前的加载器
    2. 如果不能，就抛出异常，通知其子加载器 加载 该类
  4. 重复步骤3

自定义类 委托顺序： AppClassLoader --> ExtClassloader --> BootStrapClassLoader

5、沙箱安全机制.

Java安全的核心就是 Java沙箱

什么是沙箱？.

​ 沙箱是一个限制程序运行的环境。

什么是沙箱机制？.

​ 沙箱机制就是将Java代码限定在虚拟机（JVM）特定的运行范围中，并且严格限制代码对本地资源访问，通过这种措施来保证对代码的有效隔离，防止对本地系统造成破坏。

作用.

  • 主要是限制系统的资源访问

系统资源：CPU、内存、文件系统、网络。

不同一别的沙箱对这些资源的访问的限制也不一样

所有的java程序运行都可以指定沙箱，可以定制安全策略

其基本组件.

  • 字节码校验器：确保Java类文件遵循Java语言规范。这样可以帮助Java程序实现内存保护。但并不是所有的类文件都会经过字节码校验，比如核心类

  • 类装载器：其中装载器在三个方面对 Java 沙箱起作用

    • 防止恶意代码去干涉善意的代码 双亲委派机制
    • 守护了被信任的类库边界 双亲委派机制
    • 将代码归入保护域，确定了代码可以进行那些操作 ``

6、native关键字.

native 修饰的方法 都是调用 底层 C 写的代码（C语言的库），使用的栈 是 本地方法栈

通过Java本地方法接口（JNI ：Native Method Interface）进行对其进行调用

JNI作用.

  • 为了扩展Java的使用，融合不同的编程语言为Java所用，如 最初C，C++

7、程序寄存器.

线程私有

每个线程都有一个程序计数器，是线程是有的，就是一个指针，指向方法区中的方法字节码（用来存储一条指令的地址— 即将执行的指令的），在执行引擎读取下一条指令，是一个非常小的内存空间，几乎可以忽略不计。

8、方法区.

线程共享

静态方法、常量、类信息（构造方法，接口定义），运行时的常量池存在方法区中，

但是 实例变量存在堆内存中，和方法区无关

static，final，Class、Constants Pool

9、栈.

线程私有

不存在垃圾回收机制

  • 它是一种运算受限的线性表。

  • 限定仅在表尾进行插入和删除操作的线性表。

  • 这一端被称为栈顶，相对地，把另一端称为栈底。

  • 向一个栈插入新元素又称作进栈、入栈或压栈，它是把新元素放到栈顶元素的上面，使之成为新的栈顶元素；

  • 从一个栈删除元素又称作出栈或退栈，它是把栈顶元素删除掉，使其相邻的元素成为新的栈顶元素。
